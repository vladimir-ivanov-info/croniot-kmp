---
name: refactor-with-explanation
description: Refactoriza código explicando por qué mejora diseño, legibilidad y testabilidad
disable-model-invocation: true
---

Refactoriza $ARGUMENTS siguiendo este flujo:
1. Lee el código actual.
2. Señala smells concretos.
3. Explica el impacto de cada smell.
4. Propón el refactor más pequeño posible.
5. Implementa.
6. Resume:
   - qué principio se aplicó,
   - por qué el código nuevo es mejor,
   - qué patrón o heurística debo recordar.
